# Code Explainer

Provide a deep, educational explanation of code.

## Instructions

Explain: $ARGUMENTS

### Explanation Depth Levels

**Overview**: What does this code do at a high level?

**Flow**: Step-by-step execution walkthrough

**Concepts**: Programming concepts and patterns used

**Why**: Rationale behind design decisions

**Context**: How it fits in the broader system

### Cover These Aspects

1. **Purpose**: What problem does this solve?
2. **Inputs/Outputs**: What goes in, what comes out?
3. **Dependencies**: What does it rely on?
4. **Side Effects**: What external state does it modify?
5. **Patterns**: Design patterns or idioms used
6. **Trade-offs**: Why this approach vs alternatives?
7. **Gotchas**: Non-obvious behaviors or edge cases

## Output Format

```markdown
## [File/Function Name]

### Purpose
[1-2 sentence summary]

### How It Works
[Step-by-step explanation with code references]

### Key Concepts
- **[Concept]**: [Brief explanation of the pattern/technique]

### Data Flow
[Input] -> [Processing Steps] -> [Output]

### Dependencies
- [dependency]: [why it's needed]

### Example Usage
[Code example showing typical usage]

### Related Code
- [other files/functions that interact with this]
```

Adjust explanation depth based on code complexity.
